## 📍 关键公式（速查）

### 虚影位置
```typescript
dragGhost.x = localX - dragOffset.x
dragGhost.y = localY - dragOffset.y
```

### 计算 dragOffset
```typescript
dragOffset.x = initial_localX - card.x
dragOffset.y = initial_localY - card.y
```

### Global 转 Local
```typescript
const local = container.toLocal(globalPoint)
```

### Local 映射至槽位索引
```typescript
const slotIndex = Math.round(localX / 94)  // 94 = 88 + 6
```

### 卡牌宽度
```typescript
width = 88 * size + 6 * (size - 1)
```

### 合成检查
```typescript
canMerge = itemId_A === itemId_B && tier_A === tier_B
```

## 🐛 常见问题快速解决

### 虚影消失
❌ 错误：在 destroy() 前将 `dragGhost = null`  
✅ 正确：先调用 `dragGhost.destroy()`，再设为 null

### 位置偏移错误
❌ 错误：虚影不正确跟随鼠标  
✅ 正确：验证 `dragOffset = localMouse - cardOrigin`

### 高亮不跟随
❌ 错误：`onDragging` 回调未触发  
✅ 正确：检查回调绑定和 `showExternalHighlight()` 调用

### 跨容器拖拽失败
❌ 错误：使用错误的容器调用 `toLocal()`  
✅ 正确：`toLocal()` 是相对于**当前容器**的

### 合成未触发
❌ 错误：不同 itemId 或 tier  
✅ 正确：检查 `findItemCoveringSlot()` 和槽位计算

## 关键概念汇总

### 虚影 (dragGhost)
- **目的**: 拖拽期间的视觉反馈
- **类型**: 完整的 CardView 实例，alpha=0.7
- **生命周期**: pointerdown 时创建，pointerup 时销毁
- **位置**: `dragGhost.x = localX - dragOffset.x`

### dragOffset
- **目的**: 保持鼠标与卡牌的点击偏移
- **计算**: `dragOffset = initial_localX - card.x`
- **用途**: 保持虚影在鼠标下方

### 槽位网格系统
- **槽数**: 每容器 10 个 (0-9)
- **基础卡牌宽**: 88px
- **间距**: 6px
- **槽步长**: 94px (88 + 6)
- **多尺寸**: width = 88*size + 6*(size-1)

### 坐标转换
- **全局**: 屏幕空间（左上角为0,0）
- **局部**: 容器相对（随容器变化）
- **转换**: `local = container.toLocal(global)`

### Z 轴层级（渲染顺序）
- **Z0**: 背景槽位
- **Z1**: 高亮/发光层
- **Z2**: 卡牌层（+ 顶部虚影）
- **Z3**: 覆盖层（卖出提示、状态文字）

### 合成检测
- **条件**: 相同 itemId + 相同 tier + 不同槽位
- **检查**: `itemId === target.itemId && tier === target.tier`
- **结果**: 两个物品传给 `onMerge` 回调

### 跨容器拖拽
- **流程**: 棋盘 → 场景 → 储物箱（或反向）
- **视觉**: 目标容器调用 `showExternalHighlight()`
- **API**: `api.placeItem(runId, 'board', slot1, 'stash', slot2)`

## 性能指标

### 典型拖拽操作
- 虚影创建：1次实例化
- pointermove 事件：~60次/秒（桌面）
- 虚影更新：x,y 属性修改（廉价）
- 高亮更新：Graphics 重建（中等成本）

### 优化技巧
- ✅ 复用虚影（创建一次，销毁一次）
- ✅ 创建新高亮前先销毁旧的
- ✅ 使用箭头函数作为事件处理器
- ❌ 不要在每帧重建虚影
- ❌ 不要泄漏 Graphics 对象
