# [4.3] 玩家档案与战绩统计 — 实现说明

> 任务目标：用户个人档案页，记录历史战绩、最远进度、常用英雄等。

## 数据模型

### 新增 `PvpRecord`（server/src/models/PvpRecord.ts）

每场 PvP 对战落地一条明细，字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| runId | ObjectId | 所属对局 |
| userId | ObjectId | 玩家 |
| day | number | 对战发生的天数 |
| level | number | 玩家当时等级 |
| won | boolean | 胜负 |
| heroId | string | 玩家英雄 |
| opponentHeroId | string | 对手英雄 |
| opponentLevel | number | 对手等级 |
| hpLeft | number | 战斗结束剩余血量 |
| maxHp | number | 最大血量 |
| createdAt | Date | 自动时间戳 |

索引：`userId + createdAt` 倒序，支撑"近期战绩"高效查询。

### 复用已有模型

- `Run`：单局生命周期，`status` ∈ {active, finished_win, finished_lose}，`day`/`level`/`pvpWins`/`heroId` 天然记录最远进度与通关结果。
- `User`：昵称、头像、openId。

## 数据落入点

`RunService.handlePvp()` 在 `run.save()` 之后调用 `PvpRecordModel.create(...)`，复用同一方法内已计算的 `result.attackerWon` / `result.attackerHpLeft` / `opponent`，**零额外计算开销**。

## 聚合服务

`ProfileService.getProfile(userId)`：

1. 拉取该用户全部 `Run`，按 `updatedAt` 排序取最近 N 局 → `recentRuns`
2. 统计通关/失败局数 → `wins` / `losses`
3. 取 `day` 最大值 → `farthestDay`，`level` 最大值 → `bestLevel`
4. 英雄出场计数 → `favoriteHero`
5. 拉取全部 `PvpRecord` 按时间倒序：
   - 总场次 / 胜 / 负 / 胜率
   - 按 `heroId` 分组 → `heroStats`（每英雄场次、PvP 胜、PvP 负）
   - 取最近 N 场 → `recentPvp`

## API

`GET /api/profile`

- 请求头：`x-user-id`（与现有 run 路由一致的用户解析方式）
- 响应：`{ profile: PlayerProfile }`
- `PlayerProfile` 类型定义于 `shared/src/types/api.ts`，前后端共用

## 前端对接建议（未实现）

前端可在"个人中心"场景调用 `GET /api/profile`，展示：
- 顶部：昵称、头像、总 PvP 胜率、最远天数
- 英雄战绩卡片：每个英雄的出场次数 + 胜/负
- 近期对战列表：时间、英雄 vs 对手英雄、胜负、剩余血量
