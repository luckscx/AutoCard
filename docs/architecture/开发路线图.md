## 第一层：可玩性（优先级最高）

> 目标：让人愿意连续打 3 局以上

### 1. 物品体系整理
- [ ] 从 991 条大巴扎原始数据中筛选、改造，整理出 **50~80 个** 设计完善的物品
- [ ] 每个物品需明确：效果类型、端口搭配、所属英雄/通用、Tier
- [ ] 确保物品之间存在明显的**协同构筑方向**（至少每个英雄 2~3 条核心 build 路线）

### 2. 数值平衡调整
- [ ] 金币收入曲线（每日商店价格 vs 获得金币速度）
- [ ] HP 成长节奏（英雄 baseHp + hpPerLevel 是否合理）
- [ ] 怪物强度曲线（5 只是否足够？是否需要更多梯度）
- [ ] PvP 声望节奏（20 点内赢 10 场的压迫感是否合适）

### 3. 战斗效果覆盖
- [ ] 检查现有物品效果种类：毒、燃烧、减速、加速、护甲……是否覆盖足够
- [ ] 确保战斗不会因效果单一而变得无聊

---

## 第二层：内容深度

> 目标：让每局游戏都感觉不一样

### 4. 英雄差异化
- [ ] 明确每个英雄的专属物品和核心 build 方向
- [ ] 英雄技能池（skillPool）要能真正改变打法，而不只是数值微调

### 5. 扩充事件系统
- [ ] 事件数量扩充到 **15~20 个**
- [ ] 增加更多「风险/收益权衡」类事件（更有决策感）
- [ ] 避免事件只是单纯给资源，要有取舍

### 6. Roguelike 纵深
- [ ] 升级选择界面（天赋 / 技能 / 物品升级，目前是否实现待确认）
- [ ] 物品合成/进化路线设计
- [ ] 特殊 Boss 关卡或精英战

---

## 第三层：体验打磨

> 目标：让游戏看起来、听起来都舒服

### 7. 美术资源
- [ ] 确定整体视觉风格（参考大巴扎：像素 + 奇幻风）
- [ ] 物品图标（50~80 个）
- [ ] 英雄头像（4 个）
- [ ] UI 界面整体美化（背景、面板、字体）
- [ ] 战斗特效（击打、释放技能、死亡）

### 8. 音效 & 手感
- [ ] 基础 UI 音效（点击、拖拽、购买）
- [ ] 战斗音效（攻击、受伤、胜利/失败）
- [ ] 动画节奏检查（BattleScene 演出流畅度）

### 9. 新手引导
- [ ] 第一局游戏的核心操作引导
- [ ] 端口系统说明（这是最难理解的部分）

---

## 第四层：上线准备

> 目标：能让外部用户玩到

- [ ] 用户系统完善（注册/登录/密码找回）
- [ ] 排行榜 / 段位系统
- [ ] 后端部署上云（推荐：Railway / 阿里云轻量服务器）
- [ ] 移动端接入 Capacitor（iOS + Android）
- [ ] App Store / Google Play 发布流程

## 建议推进顺序

```
整理物品池（50个设计好的物品）
    ↓
跑通几场对局，调数值节奏
    ↓
确定英雄各自的 build 方向
    ↓
扩充事件到 15~20 个
    ↓
开始美术替换
    ↓
音效 & 动画打磨
    ↓
新手引导
    ↓
上线准备
```
