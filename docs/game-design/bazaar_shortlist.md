# 大巴扎筛选清单

> 来源：大巴扎 991 张原始卡牌
> 筛选时间：2026/07/19
> 目标：从 991 张原始数据中筛选出 65 张设计合理的物品，为后续游戏落地提供候选池
> 详见：[大巴扎分析.md](./大巴扎分析.md) | [物品设计清单v1.md](./物品设计清单v1.md)

---

## 筛选原则

1. **有明确机制描述** — 优先选择 description 中有具体触发条件/联动/成长数值的物品，而非仅有名称重复的占位卡
2. **有图片资源** — 优先选择已配置 image 的卡牌，便于前端渲染
3. **端口多样性** — 覆盖 damage/burn/poison/shield/heal/haste/slow/freeze/charge 全部端口类型
4. **Tier 分层** — bronze→silver→gold→diamond→legendary 逐级递减，保证游戏经济曲线
5. **英雄均衡** — 每个英雄约 8-10 张，通用约 12 张
6. **协同可组** — 每个英雄的卡牌内部有明确的 build 协同方向（如冻结+爆发、灼烧+加速等）

## 与物品设计清单v1的关系

本筛选清单基于大巴扎 991 张原始卡牌的实际数据，保留了原始卡牌的名称、端口配置和机制描述。[物品设计清单v1.md](./物品设计清单v1.md) 则是更理想化的重新设计，将原始灵感转化为 AutoCard 自有英雄体系。

两份文档的关系：
- **本清单**：原始数据筛选，保留原版机制和名称，适合作为商店系统的初始候选池
- **v1 清单**：重新设计，适配 4 英雄体系（战士/法师/盗贼/牧师），更符合 AutoCard 定位

后续迭代可将 v1 清单的设计理念融入本清单的原始数据，形成最终物品配置。
