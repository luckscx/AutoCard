📊 核心数据结构
===============

ItemConfig (卡牌配置):
  - itemId: string              (唯一 ID)
  - name: string                (显示名称)
  - description: string         (描述)
  - size: 1|2|3                 (占用格子: 小/中/大)
  - baseTier: string            (稀有度: bronze/silver/gold/diamond/legendary)
  - price: number               (金币价格)
  - cooldown: number            (冷却时间 秒)
  - ports: Port[]               (能力端口数组)
  - targetRule: TargetRule      (目标规则)
  - tags: string[]              (标签数组)
  - categories?: string[]       (分类)
  - sourceHero?: string         (来源英雄)
  - image?: string              (图片路径 ⭐)

Port (能力):
  - category: string            (分类)
  - type: string                (类型: damage/poison/burn/heal/shield/haste/charge/slow/freeze)
  - value: number               (数值)

Tier (稀有度):
  - bronze   → 0xcd7f32 (青铜)
  - silver   → 0xc0c0c0 (银)
  - gold     → 0xffd700 (黄金)
  - diamond  → 0x00ffff (钻石)
  - legendary → 0xff44ff (传奇)

💾 数据加载流程
===============
1. bazaar_items.ts 定义 BAZAAR_ITEMS 数组 (991 个 ItemConfig)
2. BAZAAR_ITEMS_MAP = Map<itemId, ItemConfig>
3. GameState.itemsMap 初始化时加载
4. CardView/ShopCardView 通过 itemId 查询配置
5. 如配置有 image 字段 → 异步加载 WebP 图片
6. imageCache 存储已加载纹理 (避免重复加载)
7. 根据 size/tier 计算卡牌尺寸和颜色

🎯 关键技术
===========
- 渲染库: Pixi.js (WebGL 2D 渲染)
- 图片格式: WebP (现代压缩格式)
- 图片加载: Assets.load<Texture>() (异步)
- 布局系统: 屏幕 4 区分 + 卡牌 3 尺寸
- 缓存机制: Map<URL, Texture> (内存缓存)
- 自适应缩放: 保持宽高比的 Sprite 缩放

📐 卡牌尺寸计算
================
基础单位: CARD_UNIT = 88px
间距: CARD_GAP = 6px
高度: CARD_H = 110px

宽度公式: cardWidth(size) = 88 * size + 6 * (size - 1)
  - size 1: 88px
  - size 2: 182px (88*2 + 6)
  - size 3: 276px (88*3 + 12)
